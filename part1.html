Czym jest HTML

Podstawowy język znaczników do tworzenia stron internetowych

  • HTML - HyperText Markup Language
  • Opisuje strukturę strony
  • Każdy z elementów HTML reprezentowany jest przez tag
  • Elementy HTML są klockami z których składa się strona
Czym jest HTML? Jest językiem znaczników do tworzenia stron internetowych. Akronim HTML tłumaczy się jako hipertekstowy język znaczników. Dokumenty HTML sładają się z elementów HTML ktore w kodzie reprezentowane są jako tag/znacznik HTML.

Dokument HTML

Tak wygląda przykładowy dokument HTML


<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>

    <h1>My First Heading</h1>
    <p>My first paragraph.</p>

  </body>
</html>
Definicja przykładowego dokumentu HTML wygląda tak jak na przykładzie. Składa się z Doctype, głównego elementu html, elementu head i elementu body. W środku head zdefiniowany jest tytuł natomiast zawartość strony definiujemy wewnątrz body w tym wypadku podstawowy nagłówek i akapit. Doctype jest tagiem który definiuje wersję dokumentu html - mówi przeglądarce w jaki sposób ma interpretować to co jest zdefiniowane poniżej. Kiedyś było kilka różnych wersji html. Był zwykły html strict, był xhtml, był dhtml. Teraz na froncie odeszło się od wspierania starych i bardzo starych przeglądarek więc używamy i polecamy html5 - tak jak w przykładzie. html jest głównym elementem w dokumencie. Występuje zawsze i zazwyczaj się go nie styluje poza pewnymi wyjątkami. head to tag który opisuje dokument. Tutaj w przykładzie jest definicja tytułu dokumentu ale tagów w head zazwyczaj jest więcej i opisują aspekty. Tag head nie jest widoczny w przeglądarce. body - ciało dokumentu - jest widoczne w przeglądarce jak i wszystkie elementy wewnątrz niego - no chyba że zdecydujemy inaczej.

Tagi

Tag składa się z nazwy elementu zamkniętego pomiędzy ostrymi nawiasami


  <tag>zawartość</tag>
  • Tagi zazwyczaj występują w parach np.: <p>...</p>
  • Pierwszy w parze nazywa sie tagiem otwierającym a drugi zamykającym
  • Tag zamykający ma / przed nazwą
  • Występują też tagi bez tagów zamykających np.: <br/>
Ciągle mówię o tagach ale co to jest? Tagi zazwyczaj występują w parach. Tag otwierający i zamykający. Tag zawsze ma zdefiniowaną nazwę. Para tagów musi posiada tą samą nazwę z czego drugi tag nazwę ma poprzedzoną sleshem. Występują też tagi bez tagów zamykających jak na przykład tutaj br lub obrazek img. Każdy z tagów może zawierać atrybuty ze standardowego zestawu atrybutów i dodatkowo atrybuty specyficzne dla danego tagu.

Canvas

przykład


<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8"/>
    <script type="application/javascript">
    function draw() {
      var canvas = document.getElementById("canvas");
      if (canvas.getContext) {
        var ctx = canvas.getContext("2d");

        ctx.fillStyle = "rgb(200,0,0)";
        ctx.fillRect (10, 10, 50, 50);

        ctx.fillStyle = "rgba(0, 0, 200, 0.5)";
        ctx.fillRect (30, 30, 50, 50);
      }
    }
    </script>
  </head>
  <body onload="draw();">
    <canvas id="canvas" width="150" height="150"></canvas>
  </body>
</html>

Skoro HTML został stworzony do opisywania zawartości (h1 to nagłówek p to akapit itp)
To nie powinniśmy używać go do stylowania elementów. Do tego służy CSS.
CSS pozwala rozdzielić style od HTML. W tej części prezentacji opowiem jak podłączyć style do html, o box modelu i o pozycjonowaniu elementów w css.

Stylowanie

Atrybut Style


  <div style="background: red;"></div>
Style CSS można zastosować na trzy różne sposoby. Można wpisywać je inline w elementy HTML w ten sposób jak na tym przykładzie. Ale tak nie robimy.

Stylowanie

Tag style w head


<!DOCTYPE html>
<html>
  <head>
    <style>
    a:link {
      color: green;
      background-color: transparent;
      text-decoration: none;
    }
    a:visited {
      color: pink;
      background-color: transparent;
      text-decoration: none;
    }
    a:hover {
      color: red;
      background-color: transparent;
      text-decoration: underline;
    }
    a:active {
      color: yellow;
      background-color: transparent;
      text-decoration: underline;
    }
    </style>
  </head>
  <body>

  <p>You can change the default colors of links</p>

  <a href="html_images.asp" target="_blank">HTML Images</a>

  </body>
</html>
Można definiować style CSS wewnątrz elementu head strony co jest lepsze od wpisywania styli bezpośrednio w elementy html ma to swoje plusy jak na przykład style lecą do przeglądarki wraz z html w jednym requescie ale lepiej nie definiować wszystkich styli tutaj ponieważ później okazuje się że plik ma milion linijek. Na froncie wykorzystujemy style w head do AboveTheFold. Style które występują powyżej określonej wysokości strony ładują się odrazu i użytkownik widzi ostylowaną stronę jeszcze przed tym jak wszystkie pliki css zostaną załadowane.

Stylowanie

Plik CSS

<link rel="stylesheet" href="css/reveal.css">
Na froncie najczęściej korzystamy z zewnętrznych styli - takich które są w zewnętrznym pliku CSS. Przedewszystkim daje to dużo wygody przy pracy i możliwość użycia styli z tego pliku na wielu stronach bez definiowania ich drugi raz.

Selektory

Nazwa tagu


h1 {
  background: white;
}
Selektor nazwy tagu który pozwala ostylować wszystkie elementy h1 na stronie. Na fronie z tego nie korzystamy.

Selektory

Nazwa klasy


.menu {
  background: white;
}
Selektory pozwalający ostylować wszystkie elementy określonej klasy (takie które mają ustawiony atrybut class="menu" w tym wypadku). Takie selektory najczęściej są wykorzystywane na froncie. Znajduje to uzasadnienie w tym że nazwę tagu można zmienić a styl elemetu nadal będzie ten sam. A co do Id

Selektory

Id elemenut


#element-1 {
  background: white;
}
Nie wykorzystujemy go w selektorach. W sensie jest taka możliwość ale nie robimy tego. Do id często podłączone są funkcjonalności w js i czasem zmieniając wygląd elementu można przypadkiem coś wyłączyć.

Selektory

Po spacji


.menu .button {
  background: red;
}
W selektorach spacja powoduje wybór elementu wewnątrz innego elementu. W tym wypadku element o klasie button wyszukiwany jest wewnątrz elementu o klasie menu. Oczywiście można robić więcej zagłębień ale na froncie staramy się robić ich jak najmniej.

Selektory

Bezpośrednie dziecko


.menu  > .button {
  background: red;
}
Znak większości działa podobnie jak spacja ale w tym wypadku button musi być bezpośrednim dzieckiem menu. Przydaje się przy wielokrotnych zagłębieniach tej samej struktury html która musi wyglądać inaczej na każdym poziomie.

Selektory

Następny element


.menu + .menu-two {
  background: red;
}
Element menu-two zostanie wyznaczony tylko w sytuacji kiedy menu poprzedza go w ramach tego samego rodzica. Często stosuje się do stylowania czechboxów lub radiobuttonów.

Selektory

Po przecinku


.menu1, .menu2 {
  background: red;
}
Przecinek pozwala na przypisanie styli do wielu selektorów. W tym wypadku menu i menu2 będzie niało czerwone tło.
